Introduction to Cloud & Infrastructure

Security in the cloud starts with understanding what's beneath it.

Network Infrastructure vs Cloud Infrastructure

Before diving into cloud concepts, it's important to distinguish between network infrastructure and cloud infrastructure. While both are foundational, they serve different purposes and operate at different levels.

Network Infrastructure

This refers to the hardware and software components that enable communication between devices. It includes:

  • Routers, switches, firewalls
  • Local networks (LAN), VLANs, VPNs
  • Routing tables, NAT, DNS, DMZ zones
  • Traffic inspection and segmentation

Network infrastructure is usually physical or virtualized on-premise, and is managed by system/network administrators.

Cloud Infrastructure

Cloud infrastructure builds on top of networking and adds abstraction layers to offer services through virtualized environments:

  • Virtual networks (VPC, VNet) created on demand
  • Compute resources (VMs, containers) provisioned instantly
  • Storage buckets and databases available via API
  • IAM controls, resource tagging, and automation pipelines

Unlike traditional infrastructure, cloud infrastructure is elastic, abstracted, and integrated with security and management tools by design.

Key Differences

  • Control: On-prem networks give full control. Cloud is managed through provider APIs.
  • Scalability: Network infra is hardware-bound. Cloud infra scales elastically.
  • Responsibility: On-prem = full responsibility. Cloud = shared responsibility.
  • Visibility: Network tools (Wireshark, SNMP) vs Cloud logs & dashboards (CloudTrail, Azure Monitor)

Cloud doesn't replace network infrastructure — it builds upon it with agility, scalability, and new security paradigms.

What is Cloud Infrastructure?

Cloud infrastructure is the combination of hardware, virtualization, networking, and storage that powers cloud computing. Major cloud providers (like AWS, Azure, and GCP) offer virtualized access to this infrastructure on-demand.

According to AWS: "Cloud computing is the on-demand delivery of IT resources over the Internet with pay-as-you-go pricing."

Cloud infrastructure includes:

  • Compute: Virtual Machines, containers, auto-scaling groups
  • Storage: Object storage (e.g., S3), block storage, archival services
  • Networking: Virtual networks (VPCs), subnets, gateways, firewalls
  • Security Services: IAM, logging, monitoring, key management

The Shared Responsibility Model

Security in the cloud is based on a shared model. The provider handles the security "of" the cloud, while the customer is responsible for the security "in" the cloud.

Provider responsibilities:

  • Physical data centers
  • Network hardware
  • Hypervisors and base infrastructure

Customer responsibilities:

  • Operating systems, applications, data
  • Access control & user management (IAM)
  • Security configurations, network segmentation

This model varies slightly between IaaS, PaaS, and SaaS services.

From On-Premise to Hybrid Environments

Many organizations operate in hybrid environments: part of the infrastructure remains on-premise (legacy servers, internal apps), while the rest migrates to the cloud.

This introduces new challenges:

  • Log collection across distributed systems
  • Unifying access control and audit mechanisms
  • Detecting lateral movement between environments

Security teams must bridge the visibility gap between traditional perimeter-based security and cloud-native controls.

Cloud Threat Surfaces

New services bring new risks. Common attack vectors in cloud environments include:

  • Misconfigurations – Public S3 buckets, overly permissive IAM roles
  • Credential leakage – Exposed API keys, secrets in code repositories
  • Unmonitored services – Unused but accessible endpoints
  • Privilege escalation – Abuse of trust relationships or role chaining
  • Insecure APIs – Lack of authentication or input validation

Defensive teams should focus on minimizing the attack surface through hardening, monitoring, and least privilege.

Blue Team Monitoring Capabilities

Cloud platforms offer native logging and detection tools. Understanding and enabling them is key to effective monitoring:

  • Identity & Access Logs – Who accessed what, and how?
  • Audit Trails – Resource changes, login events, API calls
  • Network Flow Logs – Detect suspicious traffic patterns
  • Alerts and Rules – Triggered on anomalous activity (e.g., login from new country)

Tools like AWS CloudTrail, Azure Monitor, and GCP Cloud Audit Logs are native ways to gain observability.

Core Concepts to Master

  • Virtual networking: subnets, routing tables, gateways
  • Access control: IAM roles, policies, trust relationships
  • Storage security: encryption, lifecycle, access permissions
  • Service hardening: disabling defaults, monitoring config drift
  • Cloud incident response: log correlation, role misuse detection

These concepts form the foundation of both proactive defense and incident response in the cloud.

The cloud reshapes the perimeter, but not the responsibility. Know your architecture, monitor your assets, and defend every layer.
