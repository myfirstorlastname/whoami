Bug Bounty Report

Good hacks are invisible. Good reports make them unforgettable.

Introduction

In the bug bounty world, the report is the key to recognition and reward. It’s not a formal document like in a pentest, but a concise, clear, and 100% reproducible write-up for triagers and program security teams.

Key Characteristics of a Bug Bounty Report

  • It targets technical triagers, often under time pressure
  • It must be clear, quickly verifiable, and well-structured
  • Severity can be discussed after submission
  • A strong report helps avoid duplicates (increasing reward chances)

Recommended Structure

1. Clear Title

Examples: Stored XSS on user profile page, IDOR on /api/user/export endpoint.

2. Immediate Impact

In one or two sentences: what’s at risk? Are sensitive data exposed? Can privileges be escalated?

3. Context Description

  • Which app / URL / parameter is affected?
  • What user role was used?
  • Which feature is being targeted?

4. Step-by-Step Reproduction

This section is critical: it must let the security team reproduce the bug without guessing.

  • 1. Log in with a standard account
  • 2. Go to /profile/settings
  • 3. Edit the bio field and inject JavaScript: <script>alert(1)</script>
  • 4. Visit the public profile — the alert is triggered

5. Visual POC (Screenshots / Videos)

Attach 1–2 annotated screenshots or a link to a short video (blur any sensitive data).

6. Estimated Severity

You can suggest a severity level (using OWASP or CVSS), but stay humble — the team may adjust it.

7. Remediation

Briefly explain how to fix the issue. For example:

  • Implement a Content Security Policy (CSP)
  • Sanitize and encode user input both client- and server-side
  • Verify resource ownership server-side

Best Practices for Writing

  • Write in clear English (it’s often the expected language)
  • Number steps and include visuals where relevant
  • Stay objective — no judgment or aggressive tone
  • Avoid assumptions — test and demonstrate

Common Platforms

  • HackerOne — Industry standard
  • Bugcrowd — Public and private programs
  • YesWeHack — Growing platform (French & English)
  • Intigriti — Focus on program quality
