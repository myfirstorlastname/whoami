üîç What is Privilege Escalation?

In the context of Linux systems, privilege escalation refers to the act of obtaining higher access rights than initially granted. In Red Team operations or penetration tests, this means moving from a basic shell (non-privileged user) to full root access.

These techniques are critical for understanding system misconfigurations, weak controls, or overlooked vectors that attackers may exploit. As a pentester, they are essential in post-exploitation phases.

üß± The Process: Enumerate ‚Üí Identify ‚Üí Exploit

Privilege escalation is rarely about luck. It begins with context-aware enumeration, continues with analysis of potential vectors, and finishes with a targeted, tested exploit, ideally with minimal footprint.

Below are the main categories of techniques used in Linux environments.

üß∞ Core Techniques

üîé Enumeration First

Use tools like linPEAS, pspy, or manual checks (sudo -l, id, env) to gather context.

Everything starts here. Without context, exploitation is blind.

üî• Sudo & Cron Misconfigurations

Abuse sudo permissions or insecure cron jobs to run arbitrary code as root.

Classic and reliable if configurations are lax.

üß® SUID Binary Abuse

Search for vulnerable binaries with the SUID bit set (e.g., find / -perm -4000).

Then use GTFOBins to weaponize them.

üí£ Environment & PATH Exploits

Hijack environment variables or manipulate $PATH to redirect execution to custom payloads.

Especially common in poorly written scripts.

üîê Credentials & SSH Hijacking

Look for stored passwords, SSH keys, reused creds, or exposed tokens in files like .bash_history or config.

Leverage them to switch users or pivot laterally.

‚ò£Ô∏è Plaintext Extraction

Recover sensitive data from plaintext zip archives or known file patterns (e.g., README.txt in /root).

Tools like bkcrack or strings are helpful.

‚ò¢Ô∏è Kernel Exploits

If the kernel is outdated, known exploits like DirtyPipe or DirtyCow can be used to escalate.

Verify kernel version with uname -a and cross-check on Exploit-DB.

üß™ Practice & Detection Tips

  • Use custom labs (e.g., VulnHub, TryHackMe, or HackTheBox) to test techniques safely.
  • Always validate the impact of a method in a non-production environment before using it professionally.
  • Leave minimal traces. Remove tools, clean logs, and avoid irreversible actions.

üß† Learn Before You Launch

Each technique has specific prerequisites and detection risks. Before launching an exploit:

  • Ask: What are the consequences if this fails?
  • Test on a local VM or CTF clone if unsure.
  • Respect the target scope and environment.
