🧠 About

Injection attacks happen when untrusted input is interpreted as code or control instructions by a backend component (database, template engine, OS shell, API resolver). Mastering injections is core to web pentesting: they’re frequent, high-impact, and often chain into authentication bypass, data exfiltration, or RCE.

This course gives you a beginner-friendly map of the main injection families you’ll meet in real engagements and bug bounty. Each topic below links to a dedicated page with hands-on methodology and examples.

📘 What is an Injection?

  • Source: user-controlled data (query params, JSON body, headers, cookies, files).
  • Sink: a parser/interpreter (SQL engine, templating runtime, OS shell, GraphQL resolver).
  • Vulnerability: input is embedded unsafely in commands/expressions → attacker gains control of logic or execution.

The golden rule: never mix code and data. If you must, enforce strict encoding, parameterization, and schema validation.

🎯 Beginner Playbook

  • Probe lightly: flip booleans, measure timing/length/status code changes before heavy payloads.
  • Stay in-context: match the sink (SQL vs. JSON vs. template), encode accordingly.
  • Confirm signal: reproduce with minimal payloads before chaining to impact.
  • Think in chains: injection → data access → auth bypass → file write/RCE → lateral movement.

🛡️ Defense Mindset

  • Use parameterized queries / prepared statements and strict schema validation.
  • Apply contextual encoding (SQL/JSON/shell/template) and avoid string concatenation.
  • Segment privileges (DB least privilege, read-only accounts) and monitor anomalous query patterns.

"Separate code from data, or attackers won’t."
