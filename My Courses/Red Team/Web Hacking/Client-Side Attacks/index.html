🧠 About

Client-side attacks target the user’s browser and front-end logic: how pages render, how scripts run, how cross-origin rules are enforced, and how UI flows shape user actions. Mastering them is essential for bug bounty and modern web pentesting because small UI or policy mistakes often lead to big impact like account takeover, data theft, or action forgery.

This course index maps the major client-side families and links to dedicated pages with methodology and hands-on examples.

📘 What is a Client-Side Attack?

  • Target: the browser & DOM (scripts, HTML, CSS, storage, UI), not just the server.
  • Goal: steal data (cookies/tokens), force actions (CSRF), mislead users (UI redress), or corrupt logic (prototype pollution).
  • Levers: untrusted input in the DOM, weak browser policies (CSP, SOP), and user interaction patterns.

Think in contexts: HTML, attribute, script, URL, CSS, JSON — each has different parsing and escaping rules.

🎯 Beginner Playbook

  • Map contexts: where does input land (HTML/attr/JS/URL/CSS)? Choose payloads per context.
  • Start quiet: use length/status/timing and harmless probes before full scripts or frames.
  • Abuse flows: test cross-origin behaviors, form auto-submits, and UI overlays.
  • Chain issues: XSS → steal CSRF token → perform CSRF; DOM bug → CSP bypass; prototype pollution → XSS.

🛡️ Defense Mindset

  • Enforce contextual output encoding and a restrictive CSP.
  • Require anti-CSRF tokens; set cookies with SameSite=Lax/Strict, HttpOnly, and Secure.
  • Disallow framing (X-Frame-Options: DENY or CSP frame-ancestors).
  • Sanitize/validate input used in the DOM; avoid dangerous sinks; harden client libraries against prototype pollution.
  • For LLM/agent UIs: isolate untrusted content, treat prompts as data, and validate tool calls.

✅ Key Takeaways

  • Client-side bugs turn the browser into an attack surface; context is everything.
  • Small UI/policy oversights often chain into high-impact compromises.
  • Solid hygiene (encoding, CSP, SameSite, safe DOM APIs) blocks most classes of attacks.

"Harden the browser paths, and the payloads lose power."
