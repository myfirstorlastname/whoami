üìå What Are Network Protocols?

A network protocol is a standardized set of rules that determine how data is transmitted between devices in a network. Protocols govern syntax, semantics, synchronization, and error handling across physical, transport, and application layers.

Without protocols, devices couldn‚Äôt understand or trust the communication they receive ‚Äî there would be no web, email, streaming, or VPNs.

üß± Protocols by OSI Layer

üì∂ Layer 1 ‚Äì Physical

  • Ethernet (IEEE 802.3): Electrical/optical signaling and data transmission over copper or fiber.
  • DSL / DOCSIS / Fiber Channel: Define line coding, modulation, and channel bandwidth.

üîó Layer 2 ‚Äì Data Link

  • MAC (Media Access Control): Addressing and access to shared media.
  • ARP (Address Resolution Protocol): Maps IP ‚Üí MAC within the same LAN.
  • LLDP / CDP: Device discovery protocols (vendor-neutral and Cisco-specific).
  • PPP / HDLC: Used on serial links and WAN lines.

üåç Layer 3 ‚Äì Network

  • IPv4 / IPv6: Logical addressing, fragmentation, and routing.
  • ICMP / ICMPv6: Diagnostics, error reporting (e.g., ping, traceroute).
  • OSPF / EIGRP / BGP: Dynamic routing protocols:
    • OSPF: Link-state; builds a full map of the network.
    • BGP: Border Gateway Protocol; core of the Internet routing table.
    • EIGRP: Cisco-proprietary hybrid routing.

üì¶ Layer 4 ‚Äì Transport

  • TCP: Reliable, ordered delivery. Implements congestion and flow control.
  • UDP: Lightweight, connectionless. Used for streaming, DNS, SNMP.
  • SCTP: Multi-stream, message-oriented transport protocol. Mostly used in telecom (e.g., 5G).

üì® Layer 5-6 ‚Äì Session & Presentation

  • SSL / TLS: Encrypts sessions at higher layers. TLS 1.3 is now standard.
  • NetBIOS: Legacy session management protocol (Windows networks).
  • MIME: Encodes multimedia (emails, HTTP payloads).

üåê Layer 7 ‚Äì Application

  • HTTP / HTTPS: Web communication protocols. RESTful APIs ride on top.
  • DNS: Resolves hostnames to IPs (critical for every network app).
  • SMTP / IMAP / POP3: Email protocols for sending/receiving.
  • FTP / SFTP / TFTP: File transfer protocols (plaintext vs. encrypted).
  • SSH: Secure remote access. Often used for tunneling and automation.
  • SNMP: Monitor and manage network devices.
  • DHCP: Assigns dynamic IPs, gateways, and DNS configurations to clients.
  • LDAP: Directory-based authentication (used in AD and enterprise SSO).

üß† Specialized Protocols

  • VXLAN: Overlay networking in data centers. Encapsulates Layer 2 over Layer 3.
  • GRE: Encapsulates many protocols (including IP-in-IP).
  • IPSec: Secure tunneling at IP layer. Used in VPNs.
  • mDNS: Zeroconf protocol for local device discovery (e.g., printers).
  • RADIUS / TACACS+: Centralized authentication, especially for network devices.
  • MQTT / AMQP / CoAP: Lightweight messaging protocols in IoT ecosystems.

Protocols often overlap and coexist in layered stacks. A single HTTPS request may involve:

  • Ethernet (L2)
  • IPv6 (L3)
  • TCP (L4)
  • TLS 1.3 (L5‚Äì6)

And this is just for a browser loading a webpage. Add authentication (LDAP, RADIUS), cloud firewall inspection (WAF), load balancers, and DNS resolution, and the stack becomes a labyrinth of encapsulated protocols and state machines.

üîê Protocols and Security

  • SSL stripping, downgrade attacks: Exploit protocol negotiation flaws.
  • DNS poisoning, spoofing: Breaks name resolution trust.
  • Malformed packets: Trigger parsing bugs in protocol stacks (e.g., IPv6 extension headers).
  • Protocol tunneling: Hide traffic inside allowed protocols (DNS tunneling, ICMP tunneling).
