🧠 About

This page summarizes my experience working through the Security Information and Event Management (SIEM) module of the SOC Level 1 learning path on TryHackMe.
This module focuses on mastering SIEM platforms such as Splunk and the ELK stack, learning how to write detection queries, analyze logs, and triage security incidents. It bridges theory and practice in the daily operations of a SOC analyst.

🛰️ Philosophy

"Logs are the breadcrumbs of compromise — SIEM is the map."
  • SIEM platforms allow analysts to:
    • Correlate events from multiple sources
    • Detect malicious patterns and behaviors
    • Investigate, respond, and document incidents
  • This module taught me not only the tools, but the thought process behind an effective detection pipeline.

📘 Topics and Labs Covered

From log ingestion to alert correlation

🔧 Introduction to SIEM

  • What makes a SIEM platform: log aggregation, parsing, normalization, correlation, alerting
  • Comparison of traditional vs next-gen SIEMs
  • Real-world examples of attack chains reconstructed via SIEM

🔍 ELK Stack (Elasticsearch, Logstash, Kibana)

  • Hands-on with ELK101: querying logs, filtering fields, identifying IOC matches
  • Visualizing patterns with dashboards and time-series analysis
  • Using Kibana Discover and Timeline views for incident triage

🔐 Splunk Basics

  • Introduction to Splunk interface: search, dashboards, alerting
  • Using tstats and eval for log correlation
  • Writing detection logic for login anomalies, PowerShell misuse, lateral movement

🎯 Hands-On Challenges

  • Investigating with ELK – Threat hunting based on specific indicators
  • ItsyBitsy – Correlating multiple sources to reconstruct a phishing-to-malware incident
  • Incident Handling with Splunk – Interactive triage and escalation based on alerting logic
  • Investigating with Splunk – Discovering signs of compromise and persistent access
  • Benign – Practical scenario focusing on separating false positives from real threats

🎯 Key Takeaways

  • I can now pivot through logs in Splunk and ELK efficiently
  • I understand how alerts are triggered, what logic sits behind them, and how to validate or dismiss them
  • Correlation of host and network logs is key to comprehensive analysis
  • I’m comfortable writing custom queries to hunt for known TTPs and threat behaviors
