🧠 About

This page presents my work on the Digital Forensics and Incident Response (DFIR) module of the SOC Level 1 learning path on TryHackMe.
This module dives into real-world post-compromise analysis — how to investigate infected machines, recover evidence, and understand attacker TTPs. It mixes forensic thinking with structured incident response methodology.

🛰️ Philosophy

"Don’t just ask what happened. Ask how and why."
  • DFIR teaches patience, precision, and process:
    • What data is relevant?
    • Where can evidence be found?
    • What timeline does the attacker’s activity follow?
  • This mindset complements live SOC work: we move from alert triage to root cause and full narrative reconstruction.

📘 Topics and Tools Explored

From volatile memory to disk artifacts

🧩 DFIR Foundations

  • What defines forensics vs incident response
  • Phases of IR: preparation, detection, containment, eradication, recovery
  • Data types: volatile vs non-volatile, memory vs disk, artifacts vs indicators

🖥️ Windows Forensics

  • Analyzing system artifacts:
    • Prefetch, ShimCache, AmCache
    • Registry hives (SAM, SYSTEM, NTUSER.DAT)
    • Event logs (Security, System, Application)
  • Timeline reconstruction from multiple sources

🐧 Linux Forensics

  • Working with Bash history, cron jobs, SSH configs, logs in /var/log
  • Checking persistence mechanisms and user activity

🧠 Memory Forensics

  • Using Volatility to extract:
    • Running processes and injected code
    • Network connections and open sockets
    • Command history and loaded DLLs
  • Practical malware analysis from live memory dumps

🔬 Disk and File System Analysis

  • Using Autopsy and KAPE for:
    • File recovery and integrity checks
    • Deleted files and alternate data streams
    • Correlating file system timelines

🛡️ Incident Management Tools

  • Redline – memory and host artifact triage
  • Velociraptor – live collection and remote hunting
  • TheHive – case and workflow management

🧪 DFIR Challenge Labs

  • Unattended – Windows IR and registry analysis
  • Disgruntled – Insider threat and file recovery
  • Critical – Multi-source forensic correlation
  • Secret Recipe – Reverse-engineering attacker behavior
