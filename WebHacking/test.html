🌐 Introduction to Web Hacking

Pseudonymous cybersecurity enthusiast progressing toward OSCP certification.
“I prefer stealth over spectacle.”

🧠 About

This page documents my experience with the “Introduction to Web Hacking” module from TryHackMe’s Jr Penetration Tester learning path.

Although I’m already beyond the junior level, I completed this module as part of a structured review. It offers one of the clearest hands-on introductions to web vulnerabilities and offensive logic.

📘 Module Summary

  • 🎯 Fundamentals of how HTTP and web servers operate
  • 🔍 Basic enumeration using browser tools and CLI (e.g., curl)
  • 🧪 Exploitation of common vulnerabilities:
    • Directory traversal
    • Command injection
    • File upload bypasses
    • Authentication bypass
    • Intro to SQL injection
  • 🔧 Intro to Burp Suite for request inspection and tampering

🛰️ Philosophy

"Method beats speed. Stealth beats noise."
  • 💡 Learn through iteration: Try → Fail → Learn
  • 🧭 Master ideas, not tools. Focus on what's happening behind the scenes.
  • 🛠️ Avoid automation at first. Solve manually, then scale.

🧪 What I appreciated as an advanced learner

  • Each room progressively builds skills — the structure is pedagogically sound.
  • Many challenges require analytical reasoning, not just following instructions.
  • Minimal tool reliance: much can be done with a browser or curl.
  • It surfaces common development mistakes that become entry points for attackers.

🎓 Who should try this?

  • Students entering the world of web security.
  • Developers who want to understand how their own input handling might backfire.
  • OSCP aspirants looking to master the mindset and workflow of real-world web testing.

🛠️ Personal Tips

  • ⚠️ Never trust user input. Exploit every field: POST, GET, file uploads.
  • 📡 Use Burp Suite Repeater to methodically probe behavior.
  • 🔄 Replay and tweak until the application reveals a mistake.
  • 🧪 File upload? Try polyglot extensions like .php.jpg.
  • 🧼 Look for verbose errors, status codes, and headers. They leak info.
