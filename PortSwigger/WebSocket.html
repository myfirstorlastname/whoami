WebSockets

"WebSockets create a live channel between client and server. If not secured, attackers can eavesdrop, hijack, or manipulate the stream."

Overview

WebSockets provide a persistent, full-duplex communication channel between a client and a server. They are widely used in modern applications for real-time updates such as chat systems, financial dashboards, or multiplayer games.

However, if WebSocket messages and handshakes are not properly validated and secured, they expose new attack surfaces. The following PortSwigger labs illustrate common WebSocket vulnerabilities and how attackers exploit them.

Labs

  • Manipulating WebSocket messages to exploit vulnerabilities : Injecting malicious payloads directly into WebSocket messages to exploit weaknesses in server-side processing.
  • Cross-site WebSocket hijacking : Forcing a victimâ€™s browser to establish a malicious WebSocket connection, leading to account takeover or data exfiltration.
  • Manipulating the WebSocket handshake to exploit vulnerabilities : Tampering with the initial handshake request (headers, origin checks) to bypass security controls.

Key Takeaways

  • Always validate and sanitize WebSocket messages on the server side : never trust client input.
  • Enforce strict origin checks during the handshake to prevent cross-site hijacking.
  • Apply the same authentication and authorization controls to WebSockets as you would to HTTP requests.
  • Monitor and log WebSocket traffic, real-time channels can hide attacks if left unchecked.

WebSockets expand the attack surface. Without strict validation, attackers can turn real-time features into real-time exploits.
