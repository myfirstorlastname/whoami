üß† What is CORS?

Cross-Origin Resource Sharing (CORS) is a mechanism that allows web applications to request resources from a different domain than their own. While it is essential for modern web functionality, a misconfigured CORS policy can let attackers access sensitive data or perform actions on behalf of a victim.

CORS issues often stem from overly permissive Access-Control-Allow-Origin headers, or trusting origins that can be controlled or spoofed by an attacker.

üß© Real-World Applications

CORS vulnerabilities are rarely the main entry point in an attack chain but can have significant impact when combined with other flaws such as XSS, session fixation, or weak authentication. In a real-world scenario, exploiting misconfigured CORS can allow:

  • Reading sensitive API responses from a victim‚Äôs authenticated session
  • Stealing personal data or internal configuration information
  • Bypassing same-origin policy to pivot into private APIs
  • Combining with XSS to exfiltrate data cross-domain

üß† Key Takeaways

  • Never use Access-Control-Allow-Origin: * with credentials allowed.
  • Validate and strictly define trusted origins ‚Äî avoid reflection of user input.
  • Be cautious with null origins (common in sandboxed iframes and local files).
  • Secure endpoints with proper authentication and CSRF protections, even with CORS restrictions.
