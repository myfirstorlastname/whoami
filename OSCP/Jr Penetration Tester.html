🧭 TryHackMe Learning Path: Jr Penetration Tester

Pseudonymous cybersecurity enthusiast progressing toward OSCP certification.

🧠 About

This page documents my structured progression through the Jr Penetration Tester learning path on TryHackMe. This curated path offers a solid foundation for anyone moving toward practical offensive security—and is one of the best ways to prepare for certifications like the OSCP.

Note: I have fully completed the first module (Introduction to Web Hacking) and am actively working through the rest.

🚀 Modules

🌐 Introduction to Web Hacking ✅

A structured, hands-on entry into web exploitation. From basic enumeration to bypass techniques, this is an ideal first step for aspiring ethical hackers. Highly recommended for students.

🧰 Burp Suite 🕐

This module focuses on mastering Burp Suite—one of the most powerful tools for intercepting, modifying, and testing web requests.

🌐 Network Security 🕐

Learn the principles of network architecture and how attackers discover, map, and exploit networks. This module provides the groundwork for understanding lateral movement and initial access.

🔎 Vulnerability Research 🕐

Dive into the process of identifying and understanding vulnerabilities. Learn how to analyze services, interpret CVEs, and assess real attack surfaces.

💣 Metasploit 🕐

Metasploit is a key framework in offensive security. This module covers using it for scanning, exploitation, and post-exploitation actions.

🔐 Privilege Escalation 🕐

Learn how to move from a low-privileged shell to full system control. A practical and essential skill for Red Team and OSCP-like scenarios.

🎓 Jr Penetration Tester Certification (PT1) 🕐

The final step in the path—a practical certification to validate your skills through real-world simulated environments.

🎯 Purpose

  • ✔ Consolidate foundational knowledge before diving deeper into OSCP labs
  • ✔ Maintain a structured learning trajectory with documented milestones
  • ✔ Share progress transparently for recruiters and future teammates

This learning path is not just a checkbox list. It's a sandbox to develop technical depth, reproducible workflows, and mental models that will scale to advanced engagements. Each subpage reflects what I retained and how I think technically.
