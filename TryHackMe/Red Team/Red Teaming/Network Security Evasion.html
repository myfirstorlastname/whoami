🧠 About

This page documents my completion of the Network Security Evasion module, part of the Red Teaming learning path on TryHackMe. The module introduces how modern network defenses—such as firewalls and sandboxes—work, and how attackers adapt to bypass them.

While less technical than later Red Team modules, it provides essential knowledge for operating stealthily in environments where detection systems and filtering mechanisms are deployed.

🛰️ Philosophy

"Defense dictates the rules, but attackers can always find creative detours."
  • Evasion is not about brute force—it’s about subtlety and misdirection.
  • Understanding the limitations of network defenses helps red teamers design payloads and operations that blend in.
  • Every filtering system is a trade-off between security and usability—attackers exploit this balance.

🔎 Learning Outcomes: Network Security Evasion Module

Learning how to bypass the guards at the gate of the network.

📘 Module Summary (Rooms)

  • Network Security Solutions Overview of IDS/IPS, firewalls, and sandboxes. Emphasis on detection strategies (signature-based vs. behavioral analysis) and how attackers exploit blind spots.
  • Firewalls Covered how stateful and application firewalls enforce traffic rules. Techniques for evasion included using alternate ports, tunneling traffic, fragmenting payloads, and protocol misuses.
  • Sandbox Evasion Showed how attackers detect sandbox environments (e.g., virtualized hardware, limited runtime, unusual processes) and delay or obfuscate payload execution until outside the sandbox. Examples: time delays, API calls that behave differently in sandboxes, and multi-stage payloads that activate only in real environments.

🧪 What I appreciated

  • Clear breakdown of defensive trade-offs—why strict filtering often gets relaxed in production.
  • Sandbox evasion techniques felt close to real-world malware tactics, useful for pentesting awareness.
  • The module emphasized mindset: learning to think in terms of bypassing rather than breaking.

🎓 Who I recommend this to

  • Red team beginners who want to understand the basics of bypassing firewalls and sandboxing systems.
  • Blue teamers looking to strengthen defenses by knowing attacker evasion strategies.
  • Anyone transitioning from lab pentesting to real-world engagements, where stealth is as important as exploitation.

🛠️ My Personal Tips

  • Always fingerprint the environment—knowing what firewall or sandbox is in place guides your evasion strategy.
  • For sandbox evasion, study malware analysis reports—they’re goldmines of creative techniques.
  • Never underestimate protocol abuse: HTTP smuggling, DNS tunneling, or using WebSockets often slip through filters.
