🧠 About

This page presents a structured breakdown of the "Web Hacking Fundamentals" module from TryHackMe’s Jr Penetration Tester path. As a continuation of Introduction to Web Hacking, this module deepens your understanding of how websites can be abused—and secured—by applying methodical offensive thinking. Despite being categorized as “fundamentals”, this module introduces impactful, real-world attack scenarios with low barriers to entry and high educational value.

📘 Module Summary

  • Reinforces and expands foundational knowledge:
    • HTTP request/response manipulation
    • Directory brute-forcing and fuzzing
    • Bypassing authentication mechanisms
    • Session management weaknesses
    • Practical file upload abuse and basic shell access
  • Focuses on techniques relevant to CTFs and real-world testing scenarios.
  • Encourages hands-on experimentation over theory-heavy content.

🔍 Skills Reinforced

  • Manual enumeration using Burp Suite, browser, and CLI tools
  • Authentication bypass logic (e.g., parameter tampering, weak password checks)
  • Session hijacking fundamentals
  • Discovering hidden files and directories using ffuf and dirb
  • Understanding server-side logic behind basic access controls
  • Weaponizing file uploads and gaining foothold

🧪 What I Found Valuable

  • The fuzzing section is a great entry point to using wordlists strategically.
  • The exercise on tampering with parameters highlights the importance of manual testing, even in automated contexts.
  • Exploit paths are intentionally simple yet demonstrate real-world principles clearly.
  • Excellent lab design: clean, reproducible, and minimal guesswork.

🎓 Who Should Take This Module

  • Those who completed the previous module and want to build toward shell access and deeper logic bugs.
  • Students preparing for beginner-level CTFs or certifications like eJPT or OSCP.
  • Anyone who wants to practice and internalize the fundamentals before tackling OWASP Top 10 vulnerabilities in depth.

🛠️ Tips for Learners

  • Try the manual way first before using automated tools—it sharpens your understanding of request flows.
  • Use Burp Repeater to manipulate requests and observe server-side logic changes.
  • Always check the HTTP status codes—they’re breadcrumbs that lead to useful behavior (e.g., 403 vs 401 vs 302).
  • For brute-forcing, understand what you’re looking for before running tools blindly.
