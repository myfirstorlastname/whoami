🔎 Learning Outcomes: Introduction to Web Hacking

A beginner-friendly yet technically sound introduction to real-world web exploitation

📘 Module Summary

  • This module covers the foundational concepts of web exploitation, including:
    • Understanding how web servers and clients interact
    • Basic enumeration techniques using browser and CLI tools
    • Finding and exploiting common vulnerabilities such as:
      • Directory traversal
      • Command injection
      • File upload abuse
      • Authentication bypass
      • Basic SQL injection
    • Using Burp Suite for interception and manipulation
  • While basic, it reinforces key methodology: information gathering → manual testing → exploitation → post-exploitation.

🧪 What I appreciated as an advanced learner

  • Well-paced delivery of concepts, each room builds upon the last.
  • The labs are not overly guided: the user is expected to think and test hypotheses.
  • Encourages minimal tool dependency. Many tasks can be solved with curl, browser, or basic scripting.
  • Common developer mistakes are highlighted with practical examples.

🎓 Who I recommend this to

  • University students starting cybersecurity and looking for hands-on challenges.
  • Developers aiming to understand how their applications can be misused.
  • Anyone pursuing OSCP, as it teaches a precise and methodical approach that matches exam expectations.

🛠️ My Personal Tips

  • Don't rush. Take time to test every user input field even GET parameters.
  • Burp Suite's Repeater is your best friend: replay and modify requests systematically.
  • When exploiting uploads, always check for file extensions and bypass techniques (e.g., file.php.jpg).
  • Always analyze server responses: a 403 or 500 error is a clue, not a blocker.
  • Use curl -I or curl -v to view HTTP headers and spot potential weaknesses.
