How the Web Works

1. DNS : Domain Name System

DNS is the address book of the internet. It translates human-readable domain names (like tryhackme.com) into IP addresses (like 104.21.21.45).

Key steps of a DNS lookup:

  • User types tryhackme.com into the browser.
  • The system queries the local DNS resolver.
  • If not cached, the resolver contacts a root server, then TLD server (.com), then authoritative server.
  • Eventually, the IP address is returned and the browser connects to it.

This resolution process is mostly invisible but crucial to web communication.

2. HTTP : HyperText Transfer Protocol

HTTP is the protocol used to transfer data between the client (browser) and the server (website). It defines how requests and responses are structured.

Basic HTTP request example:


GET /index.html HTTP/1.1
Host: tryhackme.com
User-Agent: Firefox

Basic HTTP response:


HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1024

<html> ... </html>

HTTP methods like GET, POST, PUT, DELETE define the intent of a request.

3. HTML : HyperText Markup Language

HTML is the language used to structure content on the web. It's interpreted by browsers to render pages visually.

Simple HTML example:


<!DOCTYPE html>
<html>
  <head>
    <title>My Web Page</title>
  </head>
  <body>
    <h1>Welcome</h1>
    <p>This is a paragraph.</p>
  </body>
</html>

HTML works in combination with CSS (styling) and JavaScript (interactivity) to build modern web applications.

4. Full Web Request Flow (Challenge Recap)

At the end of the module, TryHackMe provides an excellent step-by-step challenge: sorting the real-world flow of a typical web request from the moment you type a domain to the moment the website is displayed.

  1. Request tryhackme.com in your browser
  2. Check Local Cache for IP Address
  3. Check your recursive DNS server for Address
  4. Query Root Server to find authoritative DNS Server
  5. Authoritative DNS Server advises the IP address for the Website
  6. Request passes through a Web Application Firewall (WAF)
  7. Request passes through a Load Balancer
  8. Connect to Web Server on port 80 or 443
  9. Web Server receives the GET request
  10. Web Application talks to the Database
  11. Your browser renders the HTML into a viewable website

This ordered process helps build a mental model of how different layers (DNS, networking, web infrastructure, backend) interact in real-world web browsing.
